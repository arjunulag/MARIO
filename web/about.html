About this project

MARIO is a from-scratch deep learning project. The underlying library is built entirely on NumPy — no PyTorch, no TensorFlow — and includes a multilayer perceptron builder, manual backpropagation, four optimizers, and a Deep Q-Network agent.

This site is the playable face of that project. It's a custom side-scrolling platformer running in pure HTML5 canvas, paired with a ghost replay system. The goal is straightforward: race a learning agent through the same level, frame for frame, and watch it improve over time.

Why a ghost?

In a racing game, a ghost is a translucent recording of a previous run. You race against the recording, not the original player. It's the cleanest way to compare two policies: same level, same physics, same starting state — just different inputs.

For an RL agent, the ghost is a natural way to make training visible. Instead of staring at a reward curve, you race the model. Early ghosts stand still or fall in pits. Later ghosts beat you to the flag.

How the ghost is wired

The ghost loads a trajectory from data/ghost_sample.json. Each entry in the trajectory is a frame index plus the set of "buttons" that were held that frame. The ghost runs through the same physics engine as the player using those inputs — so the ghost is genuinely playing the level, not playing back coordinates.

To swap in a real RL trajectory, we record the agent's chosen action each frame during a successful episode, write those actions out as JSON in this same format, and drop the file into data/. The website doesn't need to change.

Trajectory format

{
  "name": "DQN agent — episode 41,820",
  "description": "Best run from training session #7",
  "level": "1-1",
  "fps": 60,
  "inputs": [
    { "frame": 0,   "duration": 9999, "keys": ["right"] },
    { "frame": 0,   "duration": 9999, "keys": ["sprint"] },
    { "frame": 64,  "duration": 14,   "keys": ["jump"] },
    { "frame": 132, "duration": 18,   "keys": ["jump"] }
  ]
}

The underlying ML stack

The repo behind this site implements a small but complete deep learning stack on NumPy:

  • Neural networks — MLPs built from a config of linear and activation layers. He init for ReLU, Xavier for sigmoid.
  • Backpropagation — manual gradient computation through the layer list, with MSE loss.
  • Optimizers — vanilla SGD, momentum, RMSProp, and Adam, all sharing one gradient interface.
  • DQN — experience replay, ε-greedy exploration, a target network with periodic hard sync, and inline Adam with gradient clipping.
